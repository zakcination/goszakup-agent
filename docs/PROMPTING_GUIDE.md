# Prompting Guide (RU/KZ)

Этот гид помогает формулировать запросы так, чтобы ответы были точными и воспроизводимыми.

## 1) Минимально достаточные параметры

Чтобы получить корректный анализ, желательно указать:
- ЕНСТРУ (код товара/услуги)
- год (2024–2026)
- регион/город поставки (если важно сравнение внутри локации)
- БИН (если анализ по конкретному заказчику/поставщику)

## 2) Шаблоны запросов

### Справедливость цены
- `Оцени адекватность цены лота № 32899983 относительно аналогичных контрактов в идентичном городе поставки`
- `Справедлива ли цена по ЕНСТРУ 331312.100.000000 в 2025 году?`

### Аномалии
- `Найди закупки с отклонением цены >30% по ЕНСТРУ 331312.100.000000 за 2025`
- `Покажи самые аномальные закупки по этому ЕНСТРУ в 2024–2026`

### Сравнение
- `Сравни закупки по ЕНСТРУ 331312.100.000000 между БИН 000740001307 и 020440003656 за 2025`

### Объемы
- `Выяви нетипичное завышение количества ТРУ по ЕНСТРУ 172213.000.000002 за 2025`
- `Покажи объемные аномалии по БИН 020440003656 за 2025 (рост >50%)`

## 2.1) Submission demo prompts (из обязательных классов ТЗ)

- `Оцени адекватность цены лота № 32899983 относительно аналогичных контрактов других ведомств в идентичном городе поставки`
- `Найди закупки с отклонением цены >30% по ЕНСТРУ 331312.100.000000 за 2024 год`
- `Выяви нетипичное завышение количества ТРУ по сравнению с предыдущими годами по ЕНСТРУ 172213.000.000002 за 2025 год`
- `История договора №24764966`
- `Что в плане, но нет договора по 100140011059 за 2026`

## 3) Полезные уточнения

- Если нужны **идентичные города**, напишите: `в идентичном городе поставки`.
- Если нужен **точный лот**, используйте формат: `лот № 1234567`.
  Примечание: поддерживается не только внутренний `lots.id`, но и вход как `source_trd_buy_id/announcement_id`.
- Для **сравнения ведомств** — всегда указывайте два БИН.
- Для **Top-K** используйте: `топ 5` или `top-k 5`.
- Для порога отклонения: `отклонение >30%`.
- Для минимальной выборки: `N>=20`.

## 3.1) Поддерживаемые фильтры поиска (Analytics API)

`POST /search` поддерживает режимы:
- `mode=market` — рыночные срезы (`market_price_stats`)
- `mode=fair_price` — оценки справедливости (`lot_fair_price_eval`)
- `mode=anomaly` — аномалии (`lot_anomalies`)
- `mode=volume` — аномалии объемов (`volume_anomalies`)

Основные фильтры:
- `enstru_code`, `unit_code`, `customer_bin`, `region_id`
- `year` или диапазон `year_from/year_to`
- `min_n` (минимальный объем выборки)
- `min_deviation` / `max_deviation` (для fair_price/anomaly)
- `min_deviation` / `max_deviation` (для fair_price/anomaly, а в mode=volume трактуются как min/max ratio)
- `verdict` и `fallback_level` (для fair_price)
- `sort_by`, `limit`

## 4) Как формируется ответ (кратко)

Система:
1) определяет тип запроса (поиск / сравнение / аналитика / аномалии)
2) извлекает параметры (ЕНСТРУ, год, регион, БИН, лот)
3) выбирает подходящий режим поиска (`market` / `fair_price` / `anomaly`) и фильтры
4) возвращает вердикт + параметры + аналитику + Top‑K примеры

## 5) Если данных недостаточно

Если выборка мала (например, N < 20), в ответе будет вердикт `insufficient_data` и пояснение причины.
