1.2 Critical Data Quality Problems
⚠
P0 — BLOCKING: enstru_code is NULL on 96,903 of 97,920 lots (99%). Without КТРУ codes, Fair Price Engine cannot group by goods category. Every price comparison will be meaningless.
⚠
P0 — BLOCKING: contract_items has only 713 rows for 11,674 contracts. That is 6% coverage. unit_price in contract_items is the primary input to Fair Price calculation. With 94% missing, median prices will be computed on biased samples.
⚠
P0 — BLOCKING: contracts.lot_id is NULL on 100% of rows (11,674). contracts.trade_method_id is NULL on 100%. This means the Plan→Lot→Contract audit trail is broken at the last link.
⚠
P1 — DEGRADED: lots.kato_delivery is NULL on 81.2% (79,514 rows). Regional price comparison (K_region) requires delivery location. Without it, all prices are pooled nationally regardless of geography.
⚠
P1 — DEGRADED: enstru_ref has 9,009 entries but ALL section/division/group_code columns are NULL. Hierarchical fallback (national if N<20 at city level) cannot work without section-level grouping.

3.2 Fair Price Engine
The Fair Price calculation requires a reference pool of comparable prices. Given the data structure, the recommended approach is:
Reference Pool Strategy
Dimension
Implementation
Grouping key
enstru_code (КТРУ) + unit_norm + region_id (KATO level-2) + fin_year
Price source
contract_items.unit_price WHERE is_price_valid = TRUE — this is the ground truth
Fallback hierarchy
City (N≥20) → Oblast (N≥20) → National (N≥20) → Insufficient data verdict
Statistics
Median + IQR. MAD when N < 10. Winsorize at p5/p95 before computing
Time adjustment
K_time = CPI(target_quarter) / CPI(baseline_quarter). Use macro_indices table (currently 3 placeholder rows — needs real Kazakhstan CPI data)
K_vol
1.0 for MVP — disabled pending volume-discount data accumulation
K_region
Derived from ratio of regional median to national median, not hand-tuned constants
Fair Price Formula (operational)
FairPrice(enstru, kato, quarter) =
  Median( unit_price WHERE enstru=enstru AND region=region AND quarter=quarter )
  × K_region   (regional_median / national_median)
  × K_time     (CPI[target] / CPI[baseline])
  × K_vol      (= 1.0, MVP placeholder)
Deviation% = (P_actual − FairPrice) / FairPrice × 100%
IF N < 20 at any fallback level → verdict = INSUFFICIENT_DATA

4. Agentic Workflow Design
The agent translates natural language questions into structured tool calls, executes them against pre-computed analytics, and verbalizes the results. The LLM never sees raw data, never generates SQL, and never invents numbers.
4.1 Request Flow (Every Query)
1
Parse Intent
LLM classifies query into one of 8 intent types (see §4.2). Extracts structured parameters: {org, enstru, period, top_k, threshold}. Fails fast if intent is unclear.
2
Build Tool Call
Intent → allowed tool from the whitelist. Parameters are validated against schema. No free-form SQL. No parameter injection possible.
3
Check Cache
Redis lookup by (tool_name, params_hash). HIT: return cached JSON immediately. MISS: proceed to execution.
4
Execute Tool
FastAPI calls parameterized PostgreSQL or DuckDB query. Returns structured JSON with numbers, IDs, and evidence links. No text generation here.
5
Enrich Context
Attach: N (sample size), method used, data freshness timestamp, top-K comparable contract IDs with portal links.
6
Verbalize
LLM receives structured JSON + narration template. Writes the answer in Russian/Kazakh. Inserts numbers programmatically. Cannot change the computed values.
7
Drill-Down Ready
Response includes follow-up tool references. User can ask 'detail on contract X' and the next query has pre-populated context.

4.5 Insufficient Data Handling
This is a first-class outcome, not an error. When N < 20 at all fallback levels, the agent returns a structured verdict rather than a guessed number.
VERDICT: INSUFFICIENT_DATA
Причина: В базе найдено только 3 договора по КТРУ 26.20.11 в Нур-Султане за 2024.
Что есть: N=3, min=180,000 ₸, max=340,000 ₸ — слишком мало для статистически надёжной оценки.
Рекомендация: Расширьте диапазон до 2023–2024 (N=11) или до всей страны (N=47).
Действие: /expand_scope enstru=26.20.11 period=2023-2024
5. Security & Data Minimisation
Specific to the agentic layer — what the LLM is and is not allowed to see:
What LLM Receives
What LLM Never Receives
Aggregated statistics (median, count, IQR)
Raw procurement text dumps
Contract IDs and portal links
Full subject profiles with emails/phones
Deviation percentages and flags
Token, API keys, internal DB credentials
Top-K comparable IDs (not full records)
Bulk data exports or unfiltered tables
Structured JSON from tool whitelist only
Direct database access or SQL generation
ℹ
Data Minimization: The LLM prompt contains only: (a) the structured tool output JSON, (b) the narration template specifying response format, and (c) the user query. No bulk data, no internal IDs beyond what is needed, no credentials.
